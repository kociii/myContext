# PRD 管理器

用于管理 `docs/prd` 目录下项目需求文档的 Skill。

## 概述

本 Skill 帮助你：
- 查看现有的 PRD 版本及其结构
- 使用统一模板创建新的 PRD 版本
- 管理跨版本的需求文档
- 维护标准化的文档格式

## 目录结构规范

PRD 按以下方式组织：

```
docs/prd/
├── v0.2.0/
│   ├── prd.md      # 产品需求文档
│   ├── plan.md     # 开发计划
│   ├── design.md   # 界面设计文档（UI/UX + ASCII 图）
│   ├── dev.md      # 技术开发文档（架构、实现）
│   └── ...         # 其他文档（auth.md、api.md 等）
├── v0.2.1/
│   └── ...
└── v0.2.2/
    └── ...
```

**文档职责划分：**
- **prd.md**: 做什么（需求、用户故事、验收标准）
- **plan.md**: 何时做（时间线、阶段、任务）
- **design.md**: 长什么样（界面结构、布局、导航、ASCII 图）
- **dev.md**: 如何实现（技术架构、数据模型、API 设计、实现细节）

## 功能

### 1. 列出 PRD 版本

当用户想查看现有版本时：

```bash
ls -la docs/prd/
```

然后展示汇总信息：
- 可用版本（语义化排序）
- 每个版本的文件数量
- 最后修改日期

### 2. 查看 PRD 结构

当用户想探索特定版本时：

```bash
find docs/prd/<version> -name "*.md" | sort
```

汇总存在的文档及其用途。

### 3. 创建新 PRD 版本

当用户想创建新版本时：

**步骤 1:** 确定版本号
- 询问用户版本号（如 v0.3.0、v1.0.0）
- 或根据现有版本建议下一个版本

**步骤 2:** 创建目录结构
```bash
mkdir -p docs/prd/<version>
```

**步骤 3:** 根据用户意图生成模板文件

**prd.md 模板：**

## 使用模式

### 模式 1：快速创建 PRD
用户："创建一个关于用户认证功能的 v0.3.0 PRD"

1. 检查现有版本以确认版本号
2. 创建目录结构
3. 生成包含认证特定模板块的 prd.md
4. 生成包含典型开发阶段的 plan.md
5. 询问是否需要 design.md 或 dev.md

### 模式 2：查看 PRD 状态
用户："显示所有 PRD 版本" 或 "docs/prd 文件夹里有什么？"

1. 列出所有版本目录
2. 展示每个版本的文件摘要
3. 高亮显示最新版本

### 模式 3：扩展现有 PRD
用户："在 v0.2.3 PRD 中添加一个功能"

1. 读取现有的 prd.md
2. 追加新需求章节
3. 如需则更新 plan.md
4. 如需界面变更则更新 design.md
5. 如需技术变更则更新 dev.md

## 最佳实践

- 始终使用语义化版本（v0.2.0, v1.0.0）
- PRD 聚焦于"做什么"和"为什么"，而非"怎么做"
- 将技术实现细节放入 dev.md
- 将时间线和任务放入 plan.md
- 将界面结构和 ASCII 设计图放入 design.md
- 使用一致的 Markdown 格式
- 在 prd.md 中包含验收标准
- 在 plan.md 中使用 ✅ 标记已完成项

## 输出格式

```

v0.2.3 (最新) - 3 个文件
  📄 prd.md   - 存储方案重构
  📄 plan.md  - 规划中
  📄 design.md - 界面设计

v0.2.2 - 5 个文件
  📄 prd.md   - 路由与页面
  📄 plan.md  - ✅ 已完成
  📄 design.md - 界面设计
  📄 auth.md  - 认证流程
  📄 dev.md   - 技术实现

...
```

创建新 PRD 时，务必：
1. 与用户确认版本号
2. 询问需要哪些文档（prd/plan/design/dev/all）
3. 获取版本目标的简要描述
4. 生成包含适当章节的模板
5. 保存文件并确认位置
